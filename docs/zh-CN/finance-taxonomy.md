# 金融分类体系

## 设计原则

采用“纳入层级 + 主研究领域 + 来源属性标签 + 多标签 + 重要性评分”的结构。

数据库和模型输出使用稳定内部 `id`；日报、推送和界面展示使用中文 `name_zh`。

正式标签使用稳定枚举，中文名称用于展示；LLM 产出的新标签先进入建议区。

推荐机制：

```text
受控词表为主
LLM 只能从已知 id 中选择
LLM 可以建议新增标签
新增标签必须人工 review 后进入 taxonomy
```

## 字段结构

每个分类/标签建议包含：

```yaml
- id: semiconductor
  name_zh: 半导体
  aliases_zh: [芯片, 集成电路, IC]
  description_zh: 半导体设计、制造、设备、材料、封测等产业链。
```

字段含义：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| `id` | 是 | 稳定内部标识，数据库和 LLM 输出使用 |
| `name_zh` | 是 | 中文展示名 |
| `aliases_zh` | 否 | 中文别名，用于规则匹配、搜索归一化、LLM 纠偏 |
| `description_zh` | 建议 | 给人和 LLM 解释分类边界 |

英文名称和英文别名可按需补充为 `name_en` / `aliases_en`。

## 来源分类

来源分类回答两个问题：

1. 这个公众号是否应该纳入金融内容管理系统。
2. 如果纳入，它主要覆盖什么研究领域。

买方/卖方不作为一级来源分类。原因是卖方研究通常横跨宏观、策略、固收、金工、行业、公司等多个研究领域；如果把“卖方研究”放在一级，会和研究领域大量重叠。当前设计是：`classification.category` 存主研究领域，`classification.tags` 存来源属性，例如 `sell_side`、`buy_side`。

当前受控来源分类：

| id | 中文名 | 说明 |
|---|---|---|
| `macro_policy` | 宏观/政策/大类资产 | 宏观、政策、央行、财政、地缘、海外宏观、大类资产 |
| `strategy` | 策略/市场观点 | A股、港股、美股等权益策略、市场风格、资金流 |
| `quant` | 金工/量化/基金产品 | 金工、量化、指数、基金研究、FOF、ETF、衍生品 |
| `fixed_income` | 固收/FICC/信用 | 利率、信用、转债、债券、FICC |
| `industry_research` | 行业研究 | 一个或多个行业的研究 |
| `company_research` | 公司研究 | 公司、个股、财报和经营变化 |
| `market_infra` | 市场基础设施/监管 | 交易所、指数公司、监管、协会 |
| `finance_thinktank` | 财经智库/高质量观点 | 金融智库、论坛、深度观点 |
| `industrial` | 产业/科技观察 | 对行业研究有帮助的产业、科技和垂直媒体 |
| `news` | 财经新闻/资讯 | 快讯、综合财经新闻 |
| `opinion_kol` | 投资KOL/个人观点 | 个人观点、投资评论 |
| `recruiting_career` | 金融招聘/职业服务 | 金融招聘、求职、实习，默认不进 digest 主池 |
| `low_signal` | 非金融/低信号内容 | 非金融、营销、活动、生活、低信息密度 |

## 纳入层级

账号级 LLM/agent 判断需要同时输出纳入层级：

| id | 中文名 | 说明 |
|---|---|---|
| `core_finance` | 核心金融 | 进入正式管理系统和 digest 主池 |
| `finance_related` | 金融相关低优先级 | 可入库，但默认低频或不进主 digest |
| `optional_extension` | 可选扩展 | 产业/科技等对投资研究有帮助，但不属于金融主池 |
| `exclude` | 排除 | 非金融、招聘营销、生活娱乐等 |
| `needs_review` | 待确认 | 证据不足或边界不清 |

## 文章分类

文章主分类回答：“这篇文章主要是什么类型？”

当前受控文章分类：

| id | 中文名 | 说明 |
|---|---|---|
| `deep_research` | 深度研究 | 具有完整框架、数据和结论的深度内容 |
| `daily_commentary` | 日常点评 | 晨会、日评、盘后点评、快评 |
| `policy_interpretation` | 政策解读 | 政策、监管、会议、文件解读 |
| `earnings_review` | 财报点评 | 财报、业绩预告、经营数据点评 |
| `industry_tracking` | 行业跟踪 | 行业供需、价格、格局、景气度变化 |
| `company_tracking` | 公司跟踪 | 公司新闻、订单、产品、经营变化 |
| `data_review` | 数据点评 | 宏观、行业、公司或市场数据点评 |
| `roadshow_notes` | 路演纪要 | 路演、电话会、专家会、调研纪要 |
| `market_signal` | 市场信号 | 风格、资金、交易结构、风险偏好 |
| `risk_event` | 风险事件 | 突发风险和负面事件 |
| `overseas_mapping` | 海外映射 | 海外产业/公司/宏观对国内资产映射 |
| `recruiting_event` | 招聘活动 | 招聘、活动、会议、直播、课程 |
| `low_signal` | 低信号内容 | 营销、重复、低信息密度 |

## 标签分组

标签回答：“它涉及哪些行业、资产、主题，以及它是什么来源属性？”

标签可以多选，但必须来自受控词表。

当前配置文件：

```text
examples/taxonomy.finance.yaml
```

主要分组：

- `source_attribute`: 来源属性，例如 `sell_side`、`buy_side`、`media`、`kol`、`recruiting`。
- `industry`: 行业。
- `asset`: 资产市场。
- `topic`: 主题。

## LLM 分类输出

LLM 输出应使用内部 id：

```json
{
  "inclusion_tier": "core_finance",
  "primary_category_id": "industry_research",
  "tag_ids": ["sell_side", "semiconductor", "ai", "a_share"],
  "source_attribute": "sell_side",
  "importance_score": 0.78,
  "push_level": "digest",
  "reason_zh": "半导体产业链更新，包含需求和价格变化。",
  "suggested_new_tags": []
}
```

展示时转换成中文：

```text
纳入层级：核心金融
主分类：行业研究
来源属性：卖方
标签：半导体、AI、A股
重要性：0.78
```

如果 LLM 认为现有标签不足，只能输出建议：

```json
{
  "suggested_new_tags": [
    {
      "name_zh": "算力租赁",
      "group": "topic",
      "reason_zh": "现有主题标签无法覆盖算力租赁商业模式。"
    }
  ]
}
```

建议标签进入人工审核，通过后再写入正式分类体系。

## 语义分类指引

账号分类按三步完成：

1. 先判断 `inclusion_tier`：是否纳入金融内容管理系统。
2. 再判断 `classification.category`：主研究领域。
3. 最后判断 `source_attribute`：卖方、买方、媒体、KOL、招聘、产品方等来源属性。

`sell_side` / `buy_side` 不作为主分类。卖方研究可能覆盖宏观、策略、固收、金工、行业、公司等多个领域，所以应把“卖方”放入 `source_attribute`，把覆盖领域放入 `classification.category`。

### 卖方识别

卖方账号通常有以下证据：

- 账号名或文章标题中出现券商/研究团队模式，例如 `中金策略`、`天风固收`、`华泰金工`、`招商电子`、`东吴交运`。
- 标题里反复出现 `【公司名 + 覆盖领域】`、`证券研究`、`研究所`、`宏观团队`、`策略团队`、`固收团队`、`金工`、`行业周报`、`公司点评`、`深度报告`、`晨会`、`早间速递` 等。
- 常见卖方机构线索包括：中金、华泰、国泰海通/国君、申万宏源/申万、广发、兴证、东吴、招商、华创、国投/安信、开源、国联民生、华福、西部、天风、光大、国盛、华源、财通、长江、国金、浙商、银河、中信/中信建投、东方、平安、东北、中银、中泰、国信、信达、西南、太平洋等。

主研究领域按覆盖词判断：

| 覆盖词 | 主研究领域 |
|---|---|
| 宏观、经济、央行、财政、货币、大类资产 | `macro_policy` |
| 策略、权益、A股、港股、美股、市场风格、金股 | `strategy` |
| 固收、债市、利率、信用、转债、FICC、REITs | `fixed_income` |
| 金工、量化、ETF、FOF、LOF、指数、基金评价、衍生品、CTA、因子 | `quant` |
| 电子、计算机、通信、医药、地产、交运、汽车、非银、银行、商贸零售、机械、新能源、化工等 | `industry_research` |

### 非卖方金融账号

- 金融数据、研究工具、基金代销、第三方基金研究、财富/基金产品平台，通常是 `finance_related` 或低优先级 `core_finance`，主领域多为 `quant`，来源属性多为 `product_provider`。
- 金融招聘、实习、内推、职业服务统一放入 `recruiting_career` + `recruiting`，默认不进核心 digest。
- 金融媒体、财经期刊、综合资讯一般是 `finance_related` + `news` 或 `opinion_kol` + `media`，除非用户明确要求纳入核心池。
- 金融 KOL/个人观点若覆盖稳定且高价值，可为 `core_finance`；覆盖杂、信号不稳定时为 `finance_related` + `opinion_kol`。

### 强排除规则

以下类型先排除，再看关键词：学校/校友/活动群、政务和公共服务、医院、体育、影视娱乐、生活消费、文化内容、本地生活、公考/事业编考试、泛科技/编程账号、消费营销账号。单篇文章提到公司、AI、经济、招聘或金融雇主时，需要结合账号简介和连续文章判断。

最新文章用于判断账号的持续编辑方向。最新文章为空、过旧、迁移通知占主导，或近期内容明显偏离账号主业时，降低置信度或进入人工审核。

分类参考示例种子位于：

```text
examples/source_classification_reference.finance.json
```

该文件只保存合成示例账号名、纳入层级、主研究领域、来源属性和标签，不保存真实用户关注列表、文章证据或本地运行元数据。

## 重要性评分

建议第一版使用 0-1 分数：

| 分数 | 含义 |
|---:|---|
| `>= 0.85` | 高优先级，应进入提醒 |
| `0.65-0.85` | 值得进入日度摘要 |
| `0.35-0.65` | 普通归档 |
| `< 0.35` | 低信号，可不推送 |

评分因素：

- 来源 tier。
- 是否涉及核心关注行业。
- 是否为政策、财报、重大风险事件。
- 是否包含具体数据、边际变化和明确结论。
- 是否只是营销、招聘、活动或重复内容。
